# 💄 Bella Forever - Catálogo Premium de Belleza y Maquillaje

¡Bienvenido al repositorio de **Bella Forever**, un catálogo digital interactivo y premium diseñado para la marca de belleza y maquillaje *Bella Forever*. Esta aplicación web ofrece una experiencia de usuario inmersiva, elegante y sofisticada (Luxury Dark Mode) para explorar una amplia gama de cosméticos de alta calidad.

## ✨ Características Principales

- 🌟 **Diseño Premium & Luxury Dark Mode:** Interfaz de usuario estilizada con una paleta de colores sofisticada en tonos Borgoña Oscuro (`#5A1623`) y Dorado (`#D7B06A`).
- ⚡ **Animaciones Fluidas con GSAP:** Transiciones suaves, efectos de hover, y micro-animaciones interactivas utilizando GreenSock Animation Platform.
- 🛍️ **Experiencia de Compra Completa:**
  - **Carrusel de Destacados:** Sección con los productos más solicitados de la marca.
  - **Filtros por Categoría:** Organización intuitiva de productos para una navegación ágil.
  - **Detalle de Producto en Modal:** Vista detallada con información, variantes de tono, fotos de alta calidad y más.
  - **Carrito de Compras Integrado:** Permite a los usuarios seleccionar sus productos favoritos y gestionar sus pedidos para enviarlos directamente por WhatsApp.
- 📱 **Diseño 100% Responsivo:** Optimizado tanto para dispositivos móviles como computadoras de escritorio.

## 🛠️ Stack Tecnológico

- **Framework:** [React 19](https://react.dev/)
- **Build Tool:** [Vite](https://vite.dev/)
- **Estilos:** Vanilla CSS (Diseño a medida con variables personalizadas)
- **Animaciones:** [GSAP](https://gsap.com/) & [@gsap/react](https://gsap.com/resources/React/)
- **Linter:** [Oxlint](https://oxc.rs/)

## 🚀 Instalación y Desarrollo Local

Para correr el proyecto localmente, asegúrate de tener instalado [Node.js](https://nodejs.org/) y [Bun](https://bun.sh/) (o npm/yarn).

1. **Clonar el repositorio:**
   ```bash
   git clone https://github.com/SantiagoPrada2005/bella-catalog.git
   ```

2. **Instalar dependencias:**
   ```bash
   bun install
   # o bien: npm install
   ```

3. **Ejecutar el servidor de desarrollo:**
   ```bash
   bun dev
   # o bien: npm run dev
   ```

4. **Compilar para producción:**
   ```bash
   bun run build
   # o bien: npm run build
   ```
